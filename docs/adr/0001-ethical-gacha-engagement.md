# ADR 0001: ガチャ導線のエンゲージメント設計を倫理的な期待感と共有に限定する

## Status

承認済み

## Context

本システムは、小諸そばのメニュー選択をガチャ化する個人用 PWA である。`docs/requirements.md` では、ワンスクリーン即ガチャ型、条件選択後に「引く」ボタンを押して結果を見る体験、同条件での「もう一回」リロールを基本構造としている。

一方で、ガチャを引くまでの動線には改善余地がある。単に便利なランダム選択ツールに留まると「今日はこれで決めよう」という勢いが弱くなるが、ギャンブルやソーシャルゲームのような継続欲求を強く刺激する設計を取り入れると、小諸そばの昼食選びという軽い個人用途から外れる。

特に、可変報酬、希少性、ニアミス、ストリーク、ソーシャル比較は、抽選体験を強める一方で依存的・強迫的な利用につながりやすい。今回のアプリでは課金もマルチユーザーも想定しないため、継続利用を最大化するよりも「迷いを軽くし、食事選択を少し楽しくする」ことを優先する。

### 解決したい課題

- 「引く」前の期待感が弱く、昼食をガチャに委ねる気分を作りきれていない。
- ガチャらしさを強めたいが、小諸そばの素朴なコンセプトを壊したくない。
- 社交性を少し持たせたいが、ランキングや比較で煽る体験にはしたくない。
- 今後の UI 改善で、依存を狙う設計と健全な楽しさの境界を明文化したい。

### 検討した選択肢

1. 現状維持: 導線改善を行わず、条件選択と抽選結果だけに留める。
2. 倫理的な期待感と共有: 短いコピー、軽い演出、結果共有などで「引く理由」を補強する。
3. 依存性を高めるガチャ設計: ニアミス、希少ランク、ストリーク、通知、連続報酬などを採用する。
4. ソーシャル比較型: 人気ランキング、みんなの結果、比較指標などで参加感を作る。

### 各選択肢の評価

| 観点 | 現状維持 | 倫理的な期待感と共有 | 依存性を高めるガチャ設計 | ソーシャル比較型 |
|------|----------|----------------------|--------------------------|------------------|
| 引きたくなる強さ | 弱い | 中程度 | 強い | 中程度 |
| コンセプト適合 | 高い | 高い | 低い | 低い |
| 実装コスト | 低い | 低いから中程度 | 中程度 | 中程度から高い |
| 倫理リスク | 低い | 低い | 高い | 中程度 |
| 個人用ツールとの相性 | 高い | 高い | 低い | 低い |
| 要件との整合 | 既存要件通り | 既存要件を補強する | 体験の方向性が変わる | マルチユーザー想定と衝突する |

## Decision

**ガチャ導線の改善では、倫理的な期待感と結果共有のみを採用し、依存性を高めるためのニアミス、希少ランク、ストリーク、偽のソーシャル比較は採用しない。**

### 1. 採用する導線改善

「引く」前後の体験は、次の範囲で強化する。

- 「引く」ボタン周辺に、昼食選びを後押しする短いコピーを置く。
- 抽選中の演出は、待ち時間を過度に伸ばさず、期待感を作る程度に留める。
- 既存の `いなり` や `大盛り` の付与は「ちょい足し」の偶然として表現する。
- 結果画面に、端末標準の共有機能を使った任意の結果共有を追加できる。

表現例:

```text
今日の一杯、札にまかせる
迷ったら、小諸に聞く
本日のちょい足し: いなり
```

共有文例:

```text
今日の小諸そばガチャは「かき揚げせいろ + いなり」でした。
```

### 2. 採用しない導線改善

次の設計は、ガチャを引きたくなる効果があっても採用しない。

- ニアミス演出
  - 例: 「あと少しで上天丼セットだった」
- 希少ランク演出
  - 例: `SSR`、`激レア`、`伝説`
- ストリークや連続ログイン報酬
  - 例: 「3日連続で引いたら特別演出」
- 通知による再訪喚起
  - 例: 「今日はまだ引いていません」
- 偽のソーシャルプルーフ
  - 例: 「みんなが引いています」「今日の人気 No.1」
- ランキングや他者比較
  - 例: 「あなたは今週 5 回引きました」

### 3. 判断基準

今後ガチャ導線の案を追加する場合は、次の基準を満たすものだけを検討する。

| 基準 | 採用できる例 | 採用しない例 |
|------|--------------|--------------|
| 自律性 | ユーザーが自分で引く、戻る、共有する | 通知や損失回避で引かせる |
| 軽さ | 昼食選びを短時間で決められる | 演出や報酬で滞在時間を伸ばす |
| 正直さ | 実際の抽選結果だけを見せる | 存在しない人気や他者行動を見せる |
| 小諸そばらしさ | 札、献立、ちょい足しのような素朴な表現 | ソーシャルゲーム風のレアリティ表現 |

## Consequences

### Positive

- ガチャを引く前の気分作りを強めつつ、個人用の昼食選択ツールという位置づけを維持できる。
- 今後の UI 改善で、採用してよい演出と避ける演出の境界が明確になる。
- 結果共有により社交性を足せるが、ランキングや比較を持ち込まずに済む。
- `docs/requirements.md` のワンスクリーン即ガチャ型、完全ランダム、個人用ツールという前提と矛盾しない。

### Negative

- 強い継続利用や滞在時間の増加は狙いにくい。
  - → 本アプリでは継続率よりも、昼食選びの摩擦を減らすことを優先する。
- 希少ランクやストリークを使うゲーム的な派手さは出せない。
  - → 小諸そばの素朴な世界観に合うコピー、演出、共有文で補う。
- 共有機能だけでは、アプリ内のソーシャルな賑わいは生まれにくい。
  - → マルチユーザー想定を追加するまでは、端末標準の共有に限定する。

### Risks

- 「期待感」の演出が増える過程で、実質的に依存性を高める方向へ寄る可能性がある。
  - → 新しい導線案は、本 ADR の「採用しない導線改善」と「判断基準」に照らして確認する。
- 結果共有文が過度に煽る表現になる可能性がある。
  - → 共有文は結果の事実と軽い感想に留め、ランキング・勝敗・希少性を含めない。
- 抽選演出が長くなり、平日利用のテンポを損なう可能性がある。
  - → 演出時間は「昼食前に何度も使っても邪魔にならない」ことを基準に調整する。

## 決めていないこと

| 項目 | 決めない理由 | いつ決めるか |
|------|--------------|--------------|
| 具体的な CTA コピー | UI 全体の文体と合わせて調整する必要があるため | 導線改善の実装時 |
| 抽選演出の秒数・モーション | 既存演出との体感確認が必要なため | ガチャ演出を実装・調整する時 |
| 共有ボタンの有無と配置 | 結果画面の情報量、画像枠、リロール導線との兼ね合いがあるため | 結果画面改善時 |
| カロリー・価格表示との関係 | 価格・カロリー表示自体が未決のため | `docs/requirements.md` の保留事項を解消する時 |
